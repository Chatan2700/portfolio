## Why I started simple

I originally wanted to build a bigger DevLog SaaS with authentication, dashboards, and database-backed content.

That idea is still interesting, but it was too much for the first version of my portfolio.

The priority right now is to launch fast, learn foundations, and build something I can actually show.

## The architecture

The first version uses:

- Next.js for routing and rendering
- TypeScript for safer project structure
- Tailwind CSS for styling
- MDX for DevLog posts

No dashboard. No database. No authentication.

## What I learned

The biggest lesson so far is that simplicity is not the same as laziness.

A smaller system is easier to finish, easier to reason about, and easier to improve later.
